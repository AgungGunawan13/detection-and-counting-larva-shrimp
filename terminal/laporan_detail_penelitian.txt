======================================================================
DOKUMENTASI DETAIL PROSES BATCH PRA-PEMROSESAN HINGGA TRAINING YOLOv8
======================================================================

Berisi kerangka dan poin-poin mentahan pendukung materi Skripsi.

----------------------------------------------------------------------
1. TAHAP EKSTRAKSI DATASET (UNZIP)
----------------------------------------------------------------------
File Script Terkait: YOLOv8\scripts\01_unzip_dataset.py
Titik Referensi Program: Folder `YOLOv8\data\raw`
Tujuan: Memastikan hierarki dataset tidak rusak dibandingkan dengan manual extract (klik kanan di OS), serta memastikan integrasi programatis otomatis terarah.

Alokasi Berjalan:
- Algoritma `zipfile` pada Python mengekstrak dataset (.zip) yang mengandung metadata format format anotasi teks YOLO (koordinat x, y, panjang, lebar boks kotak) ke dalam ekosistem sistem.
- Folder terstruktur hasil luaran di alokasikan di path baru: `YOLOv8\data\extracted\dataset`.
- Program mendeteksi "dataset.zip" -> Log menujukkan Status "[OK] Unzip: dataset.zip -> [Target Direktori]".

----------------------------------------------------------------------
2. TAHAP PEMERIKSAAN & VALIDASI STRUKTUR DATASET (DATA VERIFICATION)
----------------------------------------------------------------------
File Script Terkait: YOLOv8\scripts\02_check_dataset.py
Titik Referensi Program: File `data.yaml` dan folder sub-dataset.
Tujuan: Menjamin integritas, proporsi ukuran label tehadap ketersediaan gambar (image existence ratio), sebelum dimasukan ke algoritma bobot, agar terhindar dari Error File Not Found saat di tengah-tengah pelatihan (training).

Alokasi Kalkulasi (Counting Results):
- Program membaca root `data.yaml` dan mendeteksi pembagian kelas ber-indeks "names: {0: 'benur'}".
- Partisi Latih (Train)	: 720 citra (.jpg/png) berbanding 720 text anotasi (.txt). 
- Partisi Validasi (Val)	: 30 citra berbanding 30 text anotasi. 
- Partisi Uji (Test)	: 30 citra berbanding 30 text anotasi. 
*Keterangan Tambahan: Output terminal juga menunjukan tulisan `labels_dir exists: True`, yang menandakan folder sinkron sepenuhnya (1:1).  

----------------------------------------------------------------------
3. TAHAP PELATIHAN MODEL CASCADING LIMA VARIAN (MODEL TRAINING)
----------------------------------------------------------------------
File Script Terkait: YOLOv8\scripts\03_train_all_models.py
Tujuan Utama: Melatih saraf kecerdasan tiruan lima arsitektrur (Nano, Small, Medium, Large, Extra Large) secara berturut-turut untuk dibandingkan variansinya.

Dasar/Basis Pengaturan (Hyperparameters Base):
1. EPOCHS: 100 siklus penuh (Menghindari plateau dengan jumlah perulangan memadai namun hemat sumber daya jika mandek di tengah).
2. IMG_SIZE: 640. Skala matriks pixel dirancang optimal tanpa membebani (OOM) memori VRAM hardware. 
3. PATIENCE (EARLY STOPPING) = 20. Mekanisme anti-overfitting murni YOLOv8. Menghentikan paksa pelatihan secara otomatis jika selama 20 epoch grafik model Validation Loss (Tingkat Eror Validasi) tidak mengalami peningkatan (stagnan), sehingga hanya menyimpan bobot optimalnya.

Spesifikasi Tambahan/Trik Augmentasi Khusus Objek Skala Mikroskopis/Padat (Benur):
1. `mixup=0.15` (Augmentasi Data)
   Menggabungkan kemunculan dua gambar bertumpuk dengan opasitas spesifik (transparan). Bertujuan mendidik model untuk mengenali morfologi objek di lingkungan visual "crowded" (benur padat).
2. `copy_paste=0.2` (Augmentasi Data)
   Menggabungkan metode segmentasi acak berupa memperbanyak letak (copy-paste) benur larva asimetris ke layout latar belakang yang sepi, agar model AI lebih peka mendeteksi luasan pixel mini.
3. `degrees=45.0` (Augmentasi Fisik)
   Merotasi asymmetrik citra secara acak hingga maksimal 45° celcius (merespon gaya berenang asimetrik udang).
4. `cos_lr=True` (Modifikasi Optimizer)
   Menyetel kurva turunan nilai kecepatan konvergensi atau Learning Rate (LR) menjadi kurva halus (Cosine AnnealingLR) di epoch-epoch akhir, mencegah loncatan bobot bergelombang yang kasar.
5. `close_mosaic=15` (Finetune Augmentasi Akhir)
   Matikan teknik manipulasi "mosaik/potong 4 gambar jadi 1" saat tersisa 15 epoch terakhir, membuat jaringan syaraf konvolusional (CNN) melakukan pengerucutan "Finetuning" terhadap kondisi nyata gambar benur sesungguhnya tanpa kebingungan artifisial.

Bukti Digital Proses & Autogenerate Output YOLOv8:
Proses pelatihan dikapsulisasi pada ekosistem folder baru bernama letak direktori: `YOLOv8\outputs\runs\[nama-tipe-yolo]\train_...`
Isi file vital yang dibentuk di folder tersebut yang menjadi bukti riset/dokumentasi bab IV:
1. `results.csv`: Tabel baristik mencetak parameter Epoch berjalan, Training Box Loss, dan akurasi (mAP50) logaritma riwayat utuh dari terminal.
2. `results.png`: Visualisasi Plot grafik komprehensif, untuk perbandingan Validation Loss dan peninggian metrik akurasi seiring waktu. 
3. Rekaman bobot terkeras (Weight System) yakni `weights/best.pt`, file utama yang dipakai untuk memprediksi sistem secara Realtime (Inference).
